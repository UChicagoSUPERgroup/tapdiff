The colors and symbols indicate how the first program listed can be changed to become the second program listed.

  • Bolded text appears in one program, but not the other.
  • A red background with a "-" indicates a part of the first program listed that does not appear in the second program. That is, it is "deleted" when changing the first program into the second.
  • A green background with a "+" indicates a part of the second program listed that does not appear in the first program. That is, it is "added" when changing the first program to the second.

The colors and symbols indicate how the first program listed can be changed to become the second program listed.

  • Bolded text appears in one program, but not the other.
  • Text with a strikethrough indicates a part of the first program listed that does not appear in the second program. That is, it is "deleted" when transforming the first program to the second.
  • Text with an orange background indicates a part of the first program listed that does not appear in the second program. That is, it is "added" when transforming the first program to the second.

The interface shows how the rules in the selected programs differ.

  • If a rule in one program is identical to a rule in the other program, they would be lined up on the same row with a white background.
  • Rules with light red background (and "-" to their left) are only in the program on the left. Rules with light green background (and "+" to their left) are only in the program on the right.
    • If a rule in one program is similar to a rule in the other program, in that they differ in only one column ("if", "while", or "then"), then they are lined up on the same row. The text that differs would have darker backgrounds (darker red background and bolded text for the rule on the left, darker green background and bolded text for the rule on the right.
    • Otherwise, the rule will be on its own row (with gray stripes on the other side), and the entire rule is bolded.

The colors and symbols show the difference between the two programs by indicating how to change the Program on Left to become the Program on Right.

  • Bolded rules should be deleted from the Program on Left.
  • Bolded rules should be added to the Program on Left if they have a light green background with "+" in front.
  • If a rule in the Program on Left can be partially changed to become a rule in the Program on Right, bolded text with solid orange background marks the result in Program on Right.

The colors and symbols show the difference between the two programs. They indicate how to change the program on the left to become the program on the right.

  • Text with an orange background indicates the parts of each program that does not appear in the other program.

The interface shows properties that apply to either one or both of the programs. A property can be:

  • An event will never happen.
  • An event will only or never happen while some states are all true, e.g. X will never happen while Y is true and Z is true.
  • Two or more states will always or never occur together, e.g. X and Y and Z will never all be true at the same time (which implies that two of them can be true at the same time, but not all three).

If there are properties that apply to both programs, they are presented in a box in the middle of the page.

The interface asks you a series of questions about what you want the smart home to do under specific scenarios. For each question, you may select one or multiple choices. If you select multiple choices, e.g. "A" and "B", the interface will take it to mean that you want either programs that do "A" or programs that do "B" in this scenario, not programs that do both "A" and "B".

Once you have answered all the questions, an orange tally board will appear on the right to show you how well each program matches your answers. You can scroll down this tally board to see the result for all programs.

The interface shows a series of flowcharts to explain what the two programs do differently under the same scenarios.
(Note that it does not show scenarios in which the selected programs make the house do the same things.)

Read the flowchart from left to right, and use the row of labels above it to understand the stages of the flowchart.

  • The blue box in the row of labels lets you click through the flowcharts.
  • Each row of the flowchart tracks the state of a device, a person, or an aspect of the environment.
  • Darker text represents changes in state, while lighter text represents states staying the same.
  • Gray callouts represent events such as what each program makes the smart home do.

If the flowchart splits into two then the two programs cause the smart home to become different states at the end. Otherwise (even though they do different things) the programs cause the smart home to end up in the same state.
